git add

Commit

git commit -m "Message pertinent"

Une branche est un ensemble de commit => ex : A->B->C->D

Un tag de commit pointe toujours sur le dernier commit

On peut taguer un commit de manière statique

Un commit c'est :

  • Id => SHA-1
  • Les modifications
  • Commentaire du Commit ( le -m )
  • Les infos de l'auteur du commit ( nom + email )
  • Les dates
  • Liste des id des parents

Un tag c'est :

  • Intitulé
  • Pointeur => SHA-1

Pointeur HEAD

Il pointe le commit ou nous sommes actuellement.

Diff

git diff

Permet de voir les differences entre les fichiers.

pour quitter lettre "q"

Git log

Permet de voir l'historique de nos versions.

pour quitter lettre "q"

git log -n number => donne le nombre de commit à afficher

Git SHOW

Permet d'avoir plus de détail sur un commit

git show "SHA-1 ID " ou "tag"

Git checkout

change le head de commit

Faire attention d'avoir tout commit ses modifications avant

git checkout + sha-1

pour revenir au dernier commit

git checkout master/main/le_nom_de_la_branche

LES TAGs STATIQUEs

Permet de nommer plus facilement une version bien précise

git checkout master/main/le_nom_de_la_branche

ensuite git tag + nom du tag

Pour supprimmer :

git tag --delete nomdutag
