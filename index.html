git add

Commit

git commit -m "Message pertinent"

Une branche est un ensemble de commit => ex : A->B->C->D

Un tag de commit pointe toujours sur le dernier commit

On peut taguer un commit de manière statique

Un commit c'est :

  • Id => SHA-1
  • Les modifications
  • Commentaire du Commit ( le -m )
  • Les infos de l'auteur du commit ( nom + email )
  • Les dates
  • Liste des id des parents

Un tag c'est :

  • Intitulé
  • Pointeur => SHA-1

Pointeur HEAD

Il pointe le commit ou nous sommes actuellement.

Diff

git diff

Permet de voir les differences entre les fichiers.

pour quitter lettre "q"

Git log

Permet de voir l'historique de nos versions.

pour quitter lettre "q"

git log -n number => donne le nombre de commit à afficher

Git SHOW

Permet d'avoir plus de détail sur un commit

git show "SHA-1 ID " ou "tag"

Git checkout

change le head de commit

Faire attention d'avoir tout commit ses modifications avant

git checkout + sha-1

pour revenir au dernier commit

git checkout master/main/le_nom_de_la_branche

LES TAGs STATIQUEs

Permet de nommer plus facilement une version bien précise

git checkout master/main/le_nom_de_la_branche

ensuite git tag + nom du tag

Pour supprimmer :

git tag --delete nomdutag

DEPOT DISTANT

Connaitre le dépot distant

git remote -v

PUSH mon projet

git push -u origin main

Push les Tags

De base git ne push pas les tags

git push origin "nom du tag"

Pour tous les tags

git push --tags

Git pull

Permet de récuperer les données sur un serveur distant

Git blame

permet de fliquer un fichier et connaitre le commit et l'utilisateur qui l'a modifier

git blame + nom du fichier

Gitignore

touch .gitignore

Les Stash

On veux changer de version avec checkout soit on commit soit on stash nos modifications ou pendant un merge

git stash list, git stash show + index du stash

ajouter les modif git stash pop + index du stash

Branche

git branche + nom => créer une branche à partir du commit actuel

git branch => list de toutes les branches

push la branch

git push --set-upstream origin brancheA => pour que github créer la branche local sur le dépot distant

Ajout pour la brancheA pour un autre dev qui ne l'a pas

Meme si vous git pull, la branche ne sera pas la car git n'upload pas les branche que vous n'utilisez pas

Apres le pull => on peut faire un git branch -a pour voir les branches local et remote

Apres le git pull faire un git branch + nom de la branche choisi pour que git télécharge la branche dans le dossier local.
